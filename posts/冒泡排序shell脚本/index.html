冒泡排序shell脚本

2020年9月2日
| 阅读

上周接到IBM实习面试通知，下周二面试加笔试，昨天刚收到邮件， 要求在面试前完成一道编程题目，是Linux 下的shell脚本，以前看过一点 bash 的语法，但是没有深入的去实践，昨天切换到Linux下面搞了一天，到现在才完成这个小程序，这个过程中学到很多东西，越发的感觉，一个程序，可以让你学到很多很多东西，各种各样的。

这个程序主要学到：

1）Linux下，fg bg命令用来切换使进程在前台和后台运行，一开始我打开gedit编辑器，因为ctrl+Z使得程序进入后台，我还以为gedit打不开了，程序错误什么，后来百度一查，才知道真相，jobs命令可以查询进程，然后fg 就可以使程序前台运行。

2）Bash不分类型，不像c/c++有整型，字符串什么的,bash全部是字符串，所以程序一开始我还再找类型，第二点，bash脚本的数组支持非常的灵活，c/c++中数组必须是固定，而且是连续的，bash中的数组，灵活的几乎没有章法而言，支持动态增加。

3）今天早上写完的时候，程序对于以为整数的支持完全可以运行，但是对于两位以上就会有问题，百思不得其解，今天又搞了一天，还是没有解答，找了大牛来，终于帮忙搞定，当然原因还是不知道，可能是因为bash 中 “>” shell解释为重定向，而不是比较大小，但是使用-gt的话就完美运行了，就是这么一个小东西纠结了2天，不过现在蛮有成就感。

#!/bin/bash swap() { #交换数组的两个元素 local temp=${integer[$1]} integer[$1]=${integer[$2]} integer[$2]=$temp return } echo “Please input some integer which you want to be sorted” read -a integer #echo “integers = ${integer[@]}” integerNum=${#integer[@]} #数组元素个数 #echo “integers length = $integerNum” #count=1 for (( i = $integerNum - 1 ; i > 0 ; i– )) do for (( j = 0 ; j < i ; j++ )) do a=${integer[$j]} b=${integer[$((j+1))]} if [ “$a” -gt “$b” ]; then swap $j $((j+1)) fi done # echo “$count: ${integer[@]}” # let “count += 1” done echo echo “${integer[@]}”
