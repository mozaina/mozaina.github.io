Gitshell上传文件到github

2020年9月2日
| 阅读

安装之类的我就不说了。。。。

1.直接打开git shell

然后把输入：cd [你的要管理的目录] –[]这个括号不要写

按下回车就到了这个目录下

2.初始化，就是在这个文件夹下面添加监听

输入：git init –这样的话你的github的文件夹下面就会出现

说明初始化成功！

3.然后直接把你要上传到github上的文件拖到github文件夹下面

然后输入 git status –这个是查看状态 会出现你刚拖进来的文件夹或者文件

上面是刚添加的文件

4.然后git add 你的文件的名字 –如果是文件夹就写文件夹的名称，如果是文件就写文件名（加后缀）

5.然后 git commit -m “ 这里写这个文件的注释”

6.然后就要上传到你的github上了

打开你的github网页。新建仓库

在git shell里输入：

git remote add origin https://github.com/这里输入你github用户名/你的仓库名.git

然后回车：输入下一行命令 git push -u origin master

如果出现这个，那么恭喜你，你成功上传了！

还有一个问题：就是关于文件的大小！因为github的默认大小是4m

如果你的文件大于4m，那么你就不能成功，

会出现这个：fatal: fatal: sha1 file “ write error: Broken pipe The remote end hung up unexpectedly error: failed to push some refs to’git@github.com:static- mkk/fisrt_small_game.git’

解决办法为：

    git config http.postBuffer 52428800  


    这里可以写的更大些.


不过，我看了文件大的原因：


        这个是eclipse自动生成的，这些不需要传！别的IDE应该也会自动生成这样的文件！


删掉这个文件夹，文件大小就会大大的减小，基本都可以传！解决办法来自stack overflow  在看一些相关的链接


最后发泄一下，fuck，这个高了我好几天~~~


最后解释一下这个：


        绿色部分为这个文件夹里新增的文件，第一个数值为新增的文件数量！


        红色部分为这个文件夹里减少的文件，最后一个红色数字为减少的文件的数量！
