Linux【Shell 简介】

从程序员的角度来看，Shell 本身是一种用C语言编写的程序，从用户的角度来看，Shell 是用户与Linux操作系统沟通的桥梁。

用户可以通过输入一条条命令，完成指定工作，也可以通过提前编写 Shell 脚本，完成批量命令的一次执行。

深入了解和熟练掌握Shell编程，是Linux 高级维护人员的专用技能之一

Shell 脚本特点

shell 脚本是用shell命令编写的可执行程序。

shell脚本保存在文本文件中。

shell脚本由shell环境解释执行的，不需编译

shell脚本文件需要具有可执行权限

Linux的Shell种类众多，每种Shell都各有特点，基本掌握其中一种即可

shell脚本编程的步骤

1、建立shell文件，使用vi编辑（推荐以*.sh结尾）

2、shell脚本的格式是固定的，内容如下：

#!/bin/sh //指定shell，脚本运行环境

#comments //注释

Your commands go here

3、设置shell脚本执行权限

chmod u+x

运行 Shell 脚本

执行运行shell程序，需执行权限

./file.sh

上述执行方式需要脚本具备执行权限，以下则不需要：

方式一：使用指定shell执行脚本：bash file.sh

方式二：使用当前shell执行脚本：.file.sh

Shell后台模式运行脚本或命令：

脚本或命令后加 &

注释：

以 # 号开头的为单行注释，shell 编程无直接的多行注释

单行注释1

单行注释2

声明变量：

shell 编程中，变量不需要提前声明，不需要指定数据类型。默认都是字符串

示例：variable_name = variable_value

变量引用：

$变量引用

#变量输出

echo $变量名

释放变量：

unset 变量名

${变量原型}

echo $variable_value

echo ${var}AA

单引号对：普通字符串

双引号对：可解释字符串中的变量

反引号对：执行并返回命令执行结果。类似$(…)

示例：

#!/bin/sh

#add user

read -p “input username:” x

useradd $x

passwd $x

echo ‘success’
