linux下的shell命令学习笔记003

linux 文件权限管理：

user 私人

group 组 -> 可以存在组内共享文件

other 非组内成员（当然也并非自己）

相关信息存储文件：

帐号： etc/passwd

密码： etc/shadow

组名： etc/group

chgrp 改变文件所属用户组

chown 修改文件所有者

chmod 修改文件权限

文件的权限，不多做解释。自己理解就好

强调一点：文件的权限针对的是它的内容

删除文件的权限不再文件本身，而在于它所在

文件夹的权限（因为该文件相当于是其对应文件

夹的内容）

——————————————

此处详细介绍一下，文件夹权限的概念：

1. r -> 表示相应的用户可以读取该文件夹下的内容

例如： ls命令

2. w -> 例如：新建，删除，重命名，移动…

3. x -> 文件夹的可只执行权限指的是：

相应的用户可以进入该目录下进行工作

例如： cd命令

——————————————

创建 文件/夹 的权限

umask

不同的用户会有一个默认值，用于在文件

默认权限下除去一些权限

文件夹默认：drwxrwxrwx

文件默认： -rw-rw-rw-

举例说明：

当前用户：umask 022

则他创建的文件权限为 -rw-r–r–

创建的文件夹的权限为 drwxr-xr-x

——————————————

linux文件种类：

[-] 普通文件，纯文本文件，二进制文件，数据格式文件

reguler，ascii，binary，data

[ d ] 目录文件 directory

[ l ] 链接文件 link

[ b ] 设备文件 -> 块设备文件 block

[ c ] 设备文件 -> 字符设备文件 character #例：鼠标，键盘

[ s ] 套接字 socket [var/run]

[ p ] 管道 FIFO，pipe

——————————————

根目录简介

usr -> UNIX Software Resource (Note: not user)

根目录下和开机过程有关的，必须和根目录放在同一个分区

etc(配置文件)， bin（重要执行文件）， dev， lib， sbin(sys..bin)

#

linux 目录配置标准：FHS -> Filesystem Hierarchy Standard

www.pathname.com/fhs/
