对shell的理解

2020年9月2日
1. 什么是shell： Shell（壳）是一种用C语言编写的程序（命令解释器），是连接用户和Unix/Linux内核的桥梁。 它的功能和windows的图形界面是一样的，只不过操作的形式不一样。 它通过建立文件的形式并行的运行多个程序，帮助用户完成很多工作； Shell即是一种命令语言，也是一种程序设计语言。 作为命令语言的时候，shell的工作形式是交互式的，用户输入一行命令，shell根据命令内容给出用户反馈信息； 作为程序设计语言时，shell的工作形式是非交互式的，在它的程序语言中，提供了许多参数和变量，并具有在高级程序语言中才具有的控制结构（if case while ），但是，shell程序不需要编译，而是从脚本文件中读取一行命令，执行一行，所以又称为解释行语言。 2. linux中shell的分类： 常见的shell有Bourne Again Shell (简称bash)、Bourne Shell（简称sh）、C-Shelll（简称csh）、Korn Shell（简称ksh）。 在文件/etc/shells中可以查看当前主机中包含的shell类型： 可以使用命令echo $SHELL 查看当前使用的shell类型： bash是 Bourne Again Shell 的缩写，是linux标准的默认shell ，它基于Bourne shell，吸收了C shell和Korn shell的一些特性。bash完全兼容sh，也就是说，用sh写的脚本可以不加修改的在bash中执行。 3. #!/bin/sh 的含义： #!/bin/bash通常都作为shell脚本的开头； 其中的‘ #!‘在此处作为一个特殊符号，后面紧接的是脚本解释器的路径，在书写正确的前提下，不同的路径代表不同类型的shell解释器。 我们发现在编写脚本时，不写#!/bin/bash这一行，脚本也同样能够执行，其实这时候系统是使用了默认的脚本解释器，查看系统的默认shell解释器，可以使用命令 echo $SHELL。 刚看书时，一段话对这个知识点解释的更精准： 上文说若不写#！行，则使用系统默认的shell解释器； 但是，下图又给出这样的的解释： 4. shell在linux操作系统中所处的位置： 图（1） 图（2） 图一的linux操作系统结构主要体现的是：用户空间和内核空间之间进行信息交换时，系统调用起着关键性的作用。 shell在这里，上通各种应用程序（以文件的形式存放在磁盘中），下通系统调用，好像胶水一样，将各种小功能程序和内核粘了起来，并在粘的节点处开了个接口，这使得各种小功能程序可以通过这个接口协同工作，从而增强了各个程序的功能。我们也知道，Shell是可编程的，它可以执行符合Shell语法的文本，往往很短的代码就可实现非常强大的功能，原因是这些shell语句通常都对系统调用做了一层封装。 图二主要展示了linux操作系统得大致构架，由硬件平台、内核、shell、类似shell的各种接口，文件系统、用户程序。 内核、shell应用程序等都依托于硬件平台。 内核由进程管理、内存管理、网络服务管理、文件系统管理和各种设备的驱动程序所组成，如下图： shell在该逻辑结构中就处在系统调用和应用程序的中间，起着命令解释器的作用。……
