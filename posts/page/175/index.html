shell 规范 可以使用 shellcheck 来检查 shell 的错误代码的警告和建议. 参照：https://www.jianshu.com/p/161618366866 变量的命名 头部引用：#!/usr/bin/env bash 注释 缩进 变量的使用 变量类型 函数 返回值 计算 分支 变量的命名 命名只能使用字母、数字，首个字符不能以数字开头。 单词均使用小写, 只读变量可使用大写。 变量中使用_来隔开字母，这是为了和数据库中的命名保持一致。 不能使用bash里的关键字，请在“附录1”中查看有哪些保留字 例如： # 禁止： local 123user="Tom" # 推荐： local user_123="Tom" # 禁止： User_Name="Tom" # 推荐： user_name="Tom" # 禁止： local userName="chenshang" # 推荐： local user_name="陈尚" # 禁止： local else="nihao" # 推荐： local nihao="nihao" 头部引用 我们往往看到大多数shell脚本的第一行是 #!/bin/bash 这句话,当然也有 #!/bin/sh、#!/usr/bin/bash,这几种写法也都算是正确,当然还有一些野路子的写法,为了避免误导这里就不示例了。本shell规约并不推荐使用上面的任何一种,而是使用 #!/usr/bin/env bash 这种。……
