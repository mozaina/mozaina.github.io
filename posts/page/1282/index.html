首先贴一下ruby使用RabbitMQ 的方法。 1.下个docker 2.拉取镜像，运行$docker pull rabbitmq:management 3.启动容器$docker run -d –name rabbitmq –publish 5671:5671 --publish 5672:5672 –publish 4369:4369 –publish 25672:25672 –publish 15671:15671 –publish 15672:15672 rabbitmq:management 4. conn = Bunny.new(:host => "IP地址", :port => "5672", :user => "guest", :password => "guest") #默认的用户名和密码，具体看你设置了 conn.start #创建连接通道 ch = conn.create_channel q = ch.queue("myQueue") #myQueue是我设置的队列名称 #向指定Topic发送消息 ch.default_exchange.publish(content, :routing_key => q.name, :payload_encoding => "string")#content是你要向队列发送的消息 conn.close 我现在在用rabbitMQ作为中间传递消息的，发送消息的是ruby on rails框架（ruby语言）写的，而接收端是springboot框架（java语言）写的，这两个之间如果传递消息必须规定编码方式等，否则就会出现乱码情况，字符和英文显示ascii码，而中文则会显示负数的ascii码（如果消费方和服务方都采用Springboot的模板就不会出现这样的问题）。解决办法如下： 发送消息那一行改为 ch.default_exchange.publish(content, :routing_key => q.name, :payload_encoding => "string",:content_encoding=>"UTF-8",:content_type=> "text/plain")#表示内容类型为文本，编码格式为UTF-8 ……
