Shell中快速跳转到特定目录

2020年9月2日
Shell中快速跳转到特定目录 在电脑上把项目代码统一放到了一个目录“~/code”下，经常需要从其它目录跳转到代码目录，总是输入目录比较麻烦，于是便想有没有更好的方式。 www.ahlinux.com 方式1: 在脚本中跳转 最开始是想在一个脚本中进行目录跳转，未果。因为脚本是在Shell的子进程中执行的，无法改变当前Shell的目录。后来得知在执行脚本前加一个“. ”，可以在当前进程中执行脚本，这样就能改变目录了。 www.ahlinux.com 1 #!/bin/bash # 跳转脚本，文件名“change_dir_script” 2 cd ‘/path/to/code’ 执行脚本 1 . change_dir_script # “.“后还有一个空格，脚本可指定完整路径，或放在PATH目录中 方式2: Shell命令别名 在Bash Shell中，可以给一些命令指定别名，且别名中还能带上命令参数。配置如下： 1 Shell配置文件位置，“~/.bashrc” 2 3 … 4 alias code=‘cd /path/to/code’ 5 … 这样便可通过“code”命令从任意目录跳转到代码目录了。 又，如果修改了“.bashrc”文件，想立即生效，可以执行： 1 source “~/.bashrc” 参考帖子：http://bbs.chinaunix.net/thread-125979-1-1.html 本文来自：Linux学习网 ……

shell中引用为什么要加

2020年9月2日
有些童鞋可能看到一些人写的脚本中，引用时用的是${变量名}，而自己尝试后发现和 $变量名 没什么区别,都是引用变量。其实它们 都是引用变量 ，确实没有区别~_~！ 不过在一些地方带有{}的引用会比较好用一些，如下： [root@rsync ~]# vim 111.sh #!/bin/bash dir=/usr/local/loyal-wang/ cat $dir/test.txt [root@rsync ~]# sh 111.sh loyal-wang 在这个简单shell脚本中，cat $dir/test.txt里变量后的/，由于变量中目录结尾有/，按理来说可以省略，但当你写成 cat $dirtest.txt时，执行脚本时会报错。 [root@rsync ~]# cat 111.sh #!/bin/bash dir=/usr/local/loyal-wang/ cat $dirtest.txt [root@rsync ~]# sh 111.sh cat: .txt: No such file or directory 其实从代码片中可以看出$引用部分已经改变，由于缺少分隔，导致引用的变量和文件名粘合在一起，导致出错。 而 ${}这种方式引用，可以避免上述的错误，使引用变量变得更加明确，以减少不必要的麻烦。……
