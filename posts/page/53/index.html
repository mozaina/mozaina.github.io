2020年9月2日
上个月试了一个程序员终极必杀技：rm -rf / 至今后遗症都没有消除，电脑空空如也，所有项目我都得重新过一遍，有些客户管理的证书，我只能厚着脸皮去要了。。。说实话，现在我在这里写“rm -rf /”的时候都不敢轻易回车！！！ 言归正传，既然mac over了，今天才想起来重新配置cocoapods，但是不顺利的是，出现了问题： 补充：已经安装的是淘宝ruby镜像https://ruby.taobao.org/ 具体配置cocoapods的步骤不说了，这里说一下这个问题吧，显然是ruby版本太低，要求 2.2.2以上 解决它吧，去把ruby版本号搞成>=2.2.2: 强烈建议 使用rvm安装ruby rvm install ruby-2.2 但是，但是竟然报错了，具体我忘记额，但是是权限和brew的问题，那么我又转向修复brew： // 清理原来brew rm -rf /usr/local/Cellar /usr/local/.git && brew cleanup // 安装 ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)" // 检查有无问题 brew doctor 具体过程见http://digitizor.com/install-homebrew-osx-el-capitan/ brew解决后，我赶紧去试试rvm install ruby-2.2,但是提示： Already installed ruby-2.2.4. To reinstall use: rvm reinstall ruby-2.2.4 OK，换成rvm reinstall ruby-2.2.4，成功,再去试试安装cocoapods sudo gem install -n /usr/local/bin cocoapods --pre 成功！ 参考：http://stackoverflow.com/questions/36402189/installing-cocoapods-on-os-x- el-capitan-error……

解决XShell6强制更新的问题

2020年9月2日
最近打开X-shell 6竟然要求强制更新，否则无法打开程序， 程序提示：”要继续使用此程序，您必须应用最新的更新或使用新版本” 但是更新又无法连接服务器……总之，程序是用不了了！！ 没办法了，网上问下度娘吧。。。。。 此处省略半小时搜索与尝试……… 最终，解决办法如下，本人亲测成功！（但是如果有条件还是希望大家支持正版，毕竟人家做个产品也不易） 1. 找到程序安装的目录：我的是C:\Program Files (x86)\NetSarang\Xshell 6，安装目录可能不同找你自己的安装目录即可 2.找到安装目录下的 nslicense.dll 文件 3.需要下载一个软件c32asm（注意：这个软件是绿色免安装的，但不要放到中文目录下，不然不能用）， 可以到我的下载里找，也可以去下面的网盘下。 4.用c32asm以16进制形式打开nslicense.dll 文件 5.搜索：7F0C81F98033E1010F8681，把86改成83，保存即可 6.将保存后的nslicense.dll 文件替换到安装目录即可，打开看看吧，可以用了 以上方法来源于网络，感谢各位网友的分享。 链接：https://pan.baidu.com/s/1AQyb0brMxfhQaZCr0LGKzQ 提取码：yecp 简码笔记，让你的代码更加简约精炼。……
