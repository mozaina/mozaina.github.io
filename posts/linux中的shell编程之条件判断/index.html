linux中的shell编程之条件判断

2020年9月2日
| 阅读

一、条件判断式语句

1. shell条件判断式语句之按文件类型判断

按照文件类型进行判断 测试选项 作用
-b 文件 判断该文件是否存在，并且是否为块设备文件（是块设备文件则为真）
-c 文件 判断该文件是否存在，并且是否为字符设备文件（是字符设备文件为真）
-d 文件 判断该文件是否存在，并且是否为目标文件（是目录为真）
-e 文件 判断该文件是否存在（存在为真）
-f 文件 判断该文件是否存在，并且是否为普通文件（是普通文件为真）
-L 文件 判断该文件是否存在，并且是否为符号链接文件（是符号链接文件则为真）
-p 文件 判断文件是否存在，并且是否为管道文件（是管道文件则为真）
-s 文件 判断文件是否存在，并且是否为非空（非空为真）
-S 文件 判断该文件是否存在，并且判断是否为套接字文件（是套接字文件则为真）

两种判断格式（都得通过echo $?命令来查看执行结果，结果为0则为真，否则为假）

  • test -e /root/install.log
  • [ -e /root/install.log ] # 注意两边要加上空格

但由上面的结果可看出，对于在/etc/sudoers中添加了执行权限的账号fz来说，如果检查/root目下的文件，必须得在命令前加上sudo,否则检查结果会不正确。

如果只进行简单的文件判断操作，可以将命令后接上&& echo yes || echo no

2、两个文件之间的判断

两个文件之间进行比较 测试选项 作用
文件1 -nt 文件2 判断文件1的修改时间是否比文件2的新（如果新则为真）
文件1 -ot 文件2 判断文件1的修改时间是否比文件2的旧（如果旧则为真）
文件1 -ef 文件2 判断文件1是否和文件2的lnode号一致，可以理解为两个文件是否为同一个文件。这个判断用于判断硬链接是很好的方法

3、两个整数之间的比较

两个整数之间的比较 测试选项 作用
整数1 -eq 整数2 判断整数1是否和整数2相等（相等为真）
整数1 -ne 整数2 判断整数1是否和整数2不相等（不相等位置）
整数1 -gt 整数2 判断整数1是否大于整数2（大于为真）
整数1 -lt 整数2 判断整数1是否小于整数2（小于位置）
整数1 -ge 整数2 判断整数1是否大于等于整数2（大于等于为真）
整数1 -le 整数2 判断整数1是否小于等于整数2（小于等于为真）

4、字符串的判断

字符串的判断 测试选项 作用
-z 字符串 判断字符串是否为空（为空返回真）
-n 字符串 判断字符串是否为非空（非空返回真）
字串1 == 字串2 判断字符串1是否和字符串2相等（相等返回真）
字串1 != 字串2 判断字符串1是否和字符串2不相等（不相等返回真）

字符串判断是否相等要注意，要给该字符串加上双引号才行，即如果判断两个字符串a,b是否相等，应该这样：

[ “$a” == “$b” ]

5、多重条件判断

多重条件判断 测试选项 作用
判断1 -a 判断2 逻辑与，判断1和判断2都成立，最终的结果才为真
判断1 -o 判断2 逻辑或，判断1和判断2有一个成立，最终的结果就为真
! 判断 逻辑非，使原始的判断式取反

实际的例子如下：

[fz@pigshow ~]$ aa=22
[fz@pigshow ~]$ [ "$aa" -gt 11 -a "$aa" -lt 33 ] && echo yes || echo no  #1
yes
[fz@pigshow ~]$ [ $aa -gt 11 -a $aa -lt 33 ] && echo yes || echo no      #2
yes
[fz@pigshow ~]$ [ aa -gt 11 -a aa -lt 33 ] && echo yes || echo no        #3
-bash: [: aa: integer expression expected
no
