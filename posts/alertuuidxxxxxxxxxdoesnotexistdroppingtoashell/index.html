ALERT! UUID=xxxxxxxxx does not exist. Dropping to a shell!

问题描述

ubuntu 18.04非正常关机后打不开了，先是黑屏显示：

ACPI Error: Need  type [Reference], found [Integer](_____ptrval____)
ACPI Error: AE_AML_OPERAND_TYPE,While resloving operands for [Store] (20180531/dswexec-427)
ACPI Error: Method parse/execution failed \_PR.CPU0._PDC, AE_AML_OPERAND_TYPE (20180531/psparse-516)

然后就是

Boot args (cat /proc/cmdline)
— Check rootdelay= (did the system wait long enough?)
— Check root= (did the system wait for the right device?)
— Missing modules (cat /proc/modules; ls /dev)
ALERT! UUID=xxxxxxxxx does not exist. Dropping to a shell!

百度以下好多问题都是:

Alert! /dev/disk/by-uuid/xxx does not exist.Dropping to a shell!

也尝试了他们提供的方法：把如图倒数第二行的uuid换/dev/sdXY或者在ro后面加上 roootdelay =120之类的，然而问题没有解决。

我在initfarms下输入 “ls /dev”发现，里边没有sda，sdab，开始以为是硬盘没装好，卸了硬盘之后在开机发现连报错的界面都进不去了…

我的解决过程

开机时插上之前的安装u盘，选择Try Ubuntu without installing，也就是进入所谓的live usb 模式。替换了原来系统中的initrd.img-4.18.0-24-generic文件（文件是从另一台好用的电脑的/boot文件夹下拷的）。具体操作如下：

live usb 模式下在终端中输入：

fdisk -l

再输入：

blkid

这里可以看到，sdb4和sdb5的uuid在本文的第一张图片中出现过。而sdb4是我安装系统时挂载/boot对应的盘符，打开在文件管理器中打开sdb4，发现里边也有一个initrd.img-4.18.0-24-generic文件。用从另一台电脑上拷过来的文件把它替换了，然后重启。开机之后就没有再进入（initfarms）界面，取而代之的是一行行滚动的[ok]xxxxxxxxxxx(好像触发了某种自我修复)：

[ok] xxxxxxxx
[ok] xxxxxxxx
[ok] xxxxxxxx
[ok] xxxxxxxx
[ok] xxxxxxxx

之后就成功开机！！！！！！
