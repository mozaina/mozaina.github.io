shell实战一模拟Jenkins任务实现简单的git自动部署

2020年9月2日
| 阅读

shell作业要求：

1、从git服务器clone和pull代码（可指定分支，http方式）

2、项目打包

3、部署到本地tomcat

4、需要支持的参数（可通过配置文件）：git项目地址、分支名称、项目本地目录

注意：脚本的话不要随意的在网页上copy运行，因为不同的操作系统会自动给它加上一个你看不到的标记，这会让新手非常的茫然，自己手动敲敲没什么坏处

从来没写过shell脚本，恶补了一番

----------------git.properties---------------
url=http://git.***************.git
branch=xjs
account=xjs
password=#密码
path=/Users/xjs/Desktop/test
tomcatPath=/Library/apache-tomcat-7.0.70/webapps

因为第一次写，真的是一步一道坎

通过读取配置文件进行参数处理

1、for pro in cat $1按行读取参数1中的输出

2、·· 反引号标记要运行的shell命令，表明不是一个简单的数据类型

3、tr “=” “ ” 表明删除字符串中的等号和空格（即切割字符串）

4、<< 重定向

-------------------Jenkins.sh-----------------
for pro in `cat $1`;
do
    entry=($(tr "=" " " <<< $pro))
    case ${entry[0]} in
    "url") url=${entry[1]};;
    "branch") branch=${entry[1]};;
    "account") account=${entry[1]};;
    "password") password=${entry[1]};;
    "path") path=${entry[1]};;
    "tomcatPath") tomcatPath=${entry[1]};;
    esac
done

得到了所有的参数之后就可以操作git了

1、[ -n $branch ] 条件，方括号和条件之间必须留空格，这条命令的意思是若branch长度为不为零则true，相反有一个 -z 即为零返回true

2、命令的组合一定要用··反引号！新手的话总是会忘记的吧，-t按时间倒序排序，|是管道，head -n 1 即取第一个文件(夹)的名字，在这里就是clone下来的新项目

cd $path
git clone ${url}
if [ -n $branch ]
then
    #获得最新产生的文件夹
    newDir=`ls -t |head -n 1`
    cd $newDir
    git pull origin ${branch}
else
    echo "不用pull"
fi
jar -cvf ${newDir}.war */
mv ${newDir}.war ${tomcatPath}

可能有不少错误，希望能够给予指正，也希望能给像我一样的新手一些帮助
