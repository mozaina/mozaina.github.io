最常用最全的命令adbshell

2020年9月2日
| 阅读

adb root

获得root权限后，获得写权限

挂载

adb remount （重新挂载系统分区，使系统分区重新可写,只针对system分区）。

adb shell mount -o remount, rw /[路径] （重新挂载系统分区，使系统分区重新可写）

搜索

adb shell cat path | findstr “搜索的String”

adb shell getprop | findstr “搜索的String”

产品信息

adb shell getprop ro.product.name 查看当前产品的名称

adb shell getprop ro.build.version.release 查看当前系统版本

adb shell getprop ro.product.cpu.abi 查看当前产品cpu架构

MAC地址: adb shell cat /sys/class/net/wlano/address

IMEI地址:adb shell settings get secure android_id

CPU信息:adb shell cat /proc/cpuinfo

更多硬件与系统属性

adb shell cat /system/build.prop

查看手机RAM

adb shell cat /proc/meminfo

查看分辨率：

adb shell dumpsys display | findstr DisplayDeviceInfo

查看电池状况

adb shell dumpsys battery (scale代表最大电量，level代表当前电量)

查看屏幕密度：

adb shell wm density (单位dpi)

显示屏参数

adb shell dumpsys window displays

屏幕截图

adb shell screencap -p /sdcard/**.png

adb shell screencap -h 帮助信息

录制屏幕

adb shell screenrecord /sdcard/**.mp4

adb shell screenrecord -help 帮助信息

自动化Monkey

adb shell monkey -p -v 500

查看系统应用

adb shell pm list packages -f

adb shell pm list packages -f | grep 应用名字部分字串

查看三方应用

adb shell pm list packages -3

//查看权限列表

adb shell pm list permissions -g -d

//查看华为相机路径
adb shell pm path com.huaweicamera

//设置本地locu.prop
adb shell setprop key value

启动activity组件
adb shell am start -n 包名/包名＋类名（-n 类名,-a action,-d date,-m MIME-TYPE,-c category,-e 扩展数据,等）。

查前台activity

adb shell dumpsys activity activities | grep mFocuseActivity

服务service

adb shell am startservice -n …

广播

adb shell am broadcast -a …

停止应用

adb shell am force-stop

清除数据

adb shell pm clear 应用包名

查应用版本

adb shell

dumpsys package 应用包名 | grep version

Log命令

adb logcat -d -v time -b all > fifle.txt

清除log缓存

adb logcat -c

通过TAG过滤

adb logcat -s “TAG”

关键字过滤

adb logcat |grep “关键字”

模拟手机按键

HOME键

adb shell input keyevent 3

返回键 ：4 ，音量+：24，，音量-：25，电源键：26

菜单键：82 ，静音键：164 ，点亮屏幕：224 熄灭屏幕：223

媒体控制：播放/暂停：85，停止86,下一首：87，上一首：88 ，恢复：126 ，暂停：127

滑动解锁

adb shell input swipe 300 1000 300 500 (起点终点坐标)

输入文本

adb shell input text hello （出现hello文本框）

附：

显示系统中全部Android平台：

android list targets

显示系统中全部AVD（模拟器）：

android list avd

启动模拟器：

emulator -avd 名称 -sdcard ~/名称.img (-skin 1280x800)

删除AVD（模拟器）：

android delete avd –name 名称

创建SDCard：

mksdcard 1024M ~/名称.img

启动DDMS：

ddms

显示当前运行的全部模拟器：

adb devices

对某一模拟器执行命令：

adb -s 模拟器编号 命令

安装应用程序：

adb install -r 应用程序.apk

获取模拟器中的文件：

adb pull

向模拟器中写文件：

adb push

缷载apk包：

adb shell

cd data/app

rm apk包

exit

adb uninstall apk包的主包名

adb install -r apk包

查看adb命令帮助信息：

adb help

在命令行中查看LOG信息：

adb logcat -s 标签名

adb shell后面跟的命令主要来自：

源码\system\core\toolbox目录和源码\frameworks\base\cmds目录。

删除系统应用：

adb remount （重新挂载系统分区，使系统分区重新可写）。

adb shell

cd system/app

rm *.apk

. 获取管理员权限：

adb root

发布端口：

你可以设置任意的端口号，做为主机向模拟器或设备的请求端口。如：
adb forward tcp:5555 tcp:8000

复制文件：
复制一个文件或目录到设备或模拟器上：
adb push
如：adb push test.txt /tmp/test.txt
从设备或模拟器上复制一个文件或目录：
adb pull
如：adb pull /addroid/lib/libwebcore.so .

查看bug报告：
adb bugreport
记录无线通讯日志：

一般来说，无线通讯的日志非常多，在运行时没必要去记录，但我们还是可以通过命令，设置记录：
adb shell
logcat -b radio

获取设备的ID和序列号：

adb get-product
adb get-serialno

访问数据库SQLite3

adb shell
sqlite3
