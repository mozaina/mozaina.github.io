一个简单的shell脚本让你告别添加头文件的烦恼

2020年9月2日
| 阅读

不知到你们有没有过这样的烦恼，每次噔噔的敲了一段代码后，准备，编译测试是否有存在，错误问题。可是这个时候啊，你发现编译器居然提示
//******************************************************************************//
warning: implicit declaration of function ‘strlen’ [-Wimplicit-function- declaration]
incompatible implicit declaration of built-in function ‘strlen’
include ‘’ or provide a declaration of ‘strlen’
//******************************************************************************//
这里假设 使用了 strlen()函数，但是没有加 头文件
这时，你心里可能没有所想的，但是，这只是一个，如果是好多个的时候。这你心里肯定就不爽了， 甚至心里大骂，为什么c不能像Java那样自动添加头文件？？？？
/////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
完美的分割线下面我来告诉一个方法怎么让他自动给你添加头文件，
其实这是要用到shell脚本—》》这个脚本的功能就是当你要新建一个新的.c文件时自动给你添加头文件，当然只能添加你自己设置好的头文件。

#!/bin/bash

if [ $# -ge 2 ]
then
echo “参数太多！”
exit
fi

if [ $# -eq 0 ]
then
vi
exit
fi

指定的文件已存在，直接编辑

if [ -e $1 ]
then
vi $1
exit
fi

//# 指定的文件不存在，判断其后缀
suffix=echo $1 | sed 's/.*\(\..*\)$/\1/'

if [ $suffix == “.c” ]
then
cp /bin/sample.c $1
vi $1
exit
fi

将这段代码考录到你的bin目录并取个名字，同时要chmod 赋予它权限777，
同时你要准备sample.c 文件，这里面就是你提前添加好的一大段头文件，

再次强调一遍，把这来个文件拷入到你的bin文件目录下，最好开启root权限，把这俩个文件的权限改成777.那个shell脚本最好也取个和vim相近的名字，比如：我取得是vii
做好一切操作后你每次，编辑新文件的时候使用那个shell脚本的名字，
我的是vii 那就 vii first.c 这个时候first.c 文件中的内容就是你刚才的sample.c中的内容，这个脚本其实就是把sample.c中的内容拷贝到你的文件下。
