解决shell脚本syntaxerrornearunexpectedtokenfi的问题

2020年9月2日
| 阅读

先列出我的shell脚本。

我是在windows编辑器里面编辑好之后再上传到linux进行执行的。

#!/bin/bash
Monitor()
{
    pgrep -x mysqld &> /dev/null
    if [ $? -ne 0 ];
    then
        echo "At time: `date` :MySQL is stop .">> /opt/service/logs/mysqlMonitor.log
        /sbin/service mysqld start
    else
        echo "MySQL server is running ."
    fi
    echo "-----------------------------------"
}
Monitor

在Linux执行后一直是报这样的错误。

mysqlmonitor.sh: line 9: syntax error near unexpected token fi' mysqlmonitor.sh: line 9:fi’

期初我以为是linux命令写错了呢，查了好多资料，很简单的语句，我敢肯定是没错的。但为何报这样的错误的呢。

我采用cat -v mysqlmonitor.sh命令查看我的代码

Monitor()^M
{^M
        pgrep -x mysqld &> /dev/null^M
        if [ $? -ne 0 ];then^M
                echo "At time: `date` :MySQLM-BM- is stop .">> /opt/service/logs/mysqlMonitor.log^M
                /sbin/service mysqld start^M
        else^M
                echo "MySQL server is running ."^M
        fi^M
        echo "-----------------------------------"^M
}^M
Monitor^M

发现在linux里面查看后自动在换行符后面带上了^M的标识，经过查阅资料知道 了在windows和linux里面的换行符是不一样的。具体也可以通过命令再次进行确认。

用vim打开文件，在命令行模式下输入

发现出来的是dos

现在通过命令：set ff=unix设置我们的代码模式

然后保存代码

再次查看发现代码编程了unix

通过cat -v **.sh查看代码正常。

这样再次执行代码就没有错误了。
