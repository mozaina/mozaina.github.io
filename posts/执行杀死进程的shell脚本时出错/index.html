初次接触Linux shell脚本，还真不知道怎么回事，编写的内容完全是套用模板，可我的就是不能正常执行。后来问同事才弄明白，原来在windows平台下，编写的shell
test.sh脚本拷贝到Linux平台下，用[vi -b] 编辑器查看脚本内容时，会发现每一行脚本末尾都会有 ^M这样的字符。必须去掉脚本才能正常执行！

杀死进程的shell脚本模板如下：

#!/bin/sh
pid=‘ps ax | grep JavaMainClassName | grep java | awk ‘{print $1}”
if[ “$pid” != “”]; then
kill -9 $pid
fi

脚本中的JavaMainClassName为java进程的关键字，一般用main方法所在的类名

给自己的建议：
需要花些时间去学习一下Shell编程，熟悉它的编码规范，语法，以便自己能够熟练的在Linux平台下部署java程序。还需多花些时间去了解Linux操作系统，去熟悉并且练习常用的一些命令。

急需了解的命令：vi awk kill
