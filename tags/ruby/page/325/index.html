脚本RubyWindows安装配置RubyOnRails

2020年9月2日
感觉Java学的差不多了，想接触下Ruby On Rails，看看它比Java WEB高效到哪里了，在Ubuntu12.10上弄了两天总是报错提示没有指向的文件sqlite3，不管怎么安装sqlite3都不行，到最后没办法只有跑到WIN7平台下试试，以下就是我搭建Ruby On Rails的过程： 首先也是需要下载ruby,在Windows下有直接安装的exe文件，可以从rubyinstaller下载到自己需要的ruby版本，我下载是rubyinstaller-2.0.0-p247，我看有的教程还说需要安装gem，我发现通过rubyinstaller-2.0.0-p247安装之后直接就有gem了，所以安装gem这一步也就省略了。安装之后你可以打开Start Command Prompt with Ruby命令窗口，查看版本号：ruby -v和gem -v 我是不明白，天朝为啥把那么多的技术网站都给墙了，淘宝给我们提供了一个镜像http://ruby.taobao.org可供我们使用,所以在通过gem install rails 之前我们需要把镜像源从国外改到淘宝提供的上去。可借助下列命令： [plain] view plaincopyprint? gem sources -a http://ruby.taobao.org/ 可以通过下列命令查看所以镜像源： [plain] view plaincopyprint? gem sources -l 同时也可以删除镜像源： gem sources --remove https://rubygems.org/ 自我感觉把这些国外被墙的镜像源去掉会节约时间，我是把所以国外的都去掉就剩下淘宝的镜像源。这个时候就可以开始安装Rails了： [html] view plaincopyprint? gem install rails -V 当我这安装到最后的时候出现了一个错误：意思就是缺少DevKit，这个时候还需要Devkit下载个DevKit。下载之后你可以解压到指定的目录中，如我的：D:/devkit。之后进入到这个目录，执行ruby dk.rb然后根据提示执行ruby dk.rb init和ruby dk.rb install。等待安装成功！ ruby dk.rb install -– Invalid configuration or no Rubies listed.
