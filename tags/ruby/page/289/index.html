基于RubyonRails开发以太坊的应用

2020年9月2日
一、 理解以太坊的网络架构 其中Node运行geth或者eth，彼此通过30303端口进行p2p的连接，其上运行的协议即俗称挖矿协议，也即共识协议，包括几个部分，广播交易或消息，同步区块等。 Node即节点，经常也称geth client或geth server。 每个node对外提供JSONRPC(8545端口)或者IPC端口，可以通过Node从而对整个区块链进行操作，如查询帐户信息，部署合约等。 以太坊的wallet就是一个通过ipc接口来操作区块链的典型应用(application)，但其并不是一个dApp（分布式app）。 二、 以太坊的dApp架构 理解了上文中的网络架构，一个以太坊的dApp架构也就比较容易理解了。 底层即blockchain层，可以选择ethereum或者testnet或者testrpc。 绿色部分根据框架选择的不同会有较大的区别，但基本原理是相通的，即框架要提供操作区块链接口的功能，最好是完整的协议栈。由于官方提供web3.js，这是一个完整的基于node.js实现的RPC协议接口，所以基于JS来实现的Web Framework框架会比较自然，比较知名的有truffle，官方推荐的是meteor。但理论上任何一个框架，只要提供与Blockchain交互的协议（即连接8545 RPC端口），都可以用于实现dApp，并且超过80%的部分是传统WEB或移动应用开发内容。 三、 环境准备 操作系统Ubuntu 16.04 AMD64 step 1: 安装rvm/ruby/rails #gpg –keyserver hkp://keys.gnupg.net –recv-keys 409B6B1796C275462A1703113804BB82D39DC0E3 #\curl -sSL https://get.rvm.io | bash #rvm install 2.3 #gem install bundler #gem install rails step2:经过上述步骤应该已经将rails 4.2.x安装至系统中，由于国内使用ruby的开发者不多，有限的群体应该都具备安装环境这项基本技能，加之本文重点不在于介绍rails的安装和使用，如果有问题可以百度或者google。检查环境，保证rails运行正确。 $ rails –v step3: 创建一个rails项目 $ rails new helloworld step4: 安装nodejs $ curl -sL https://deb.nodesource.com/setup_4.x | sudo -E bash - $ sudo apt-get install -y nodejs……
