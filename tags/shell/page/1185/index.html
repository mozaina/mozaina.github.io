所谓的位置参数（positional parameters）指的也就是shell脚本的命令行参数（command line arguments）。在shell函数里，它们同时也可以是函数的参数。各参数都由整数来命名。由于历史原因，当它超过9时，就应该用大括号把数字框起来： echo first arg is $1 echo first arg is ${10} 此外，通过特殊变量，我们还可以取得参数的总数，以及一次取得所有参数（后期会学习到） 假如我们想查找某个用户，我们可以直接使用who命令，然后再输出中自己慢慢找。这样做很麻烦也很容易出错—特别是放系统用户很多的时候。你想做的只不过是在who的输出中找到那位用户，这个时候你可以用grep命令来进行查找操作，它会列出与第一个参数（所指定的模式）匹配的每一行。 其实我们可以将命令放进脚本里，这段脚本的第一个参数就是我们要找的用户名称： 以# find user …开头的这一行是一个注释（comment）.shell会忽略由#开头的每一行，为程序加上注释绝对不会错。这样自己在以后还能够了解你在做什么以及为什么这么做。等到我们觉得程序能够运行无误时，就可以移动个人的bin目录。 这个程序还没有到达完美，如果没有给任何参数，会出现提示： 在后期我们会接触到当参数数目不符合的时候，如何采取适当的操作。……
