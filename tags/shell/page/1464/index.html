2020年9月2日
声明：本文内容为本人学习嵌入式linux所遇问题后，借鉴并结合自身情况所写，为自己所学留下痕迹，如能助人，也算报答前人之恩！ 对于任何想适当精通一些系统管理知识的人来说，掌握Shell脚本知识都是基本的，学习如何编写Shell脚本并不是一件困难的事，因为脚本可以分为很小的块，并且相对于Shell特性的操作和选项部分，我们只需要学习一小部分就可以了。Shell语法简单并且直观，编写脚本很像是在命令行上把相关命令和工具连接起来，并且只有很少的一部分需要学习。但是在学习的Shell脚本的过程中，我们还是需要注意一些东西，这对于向我们这样的初学者来说是非常重要的。详细如下： -—————————————————————————————————————————————————————————- 1、开头 Shell脚本的开头有写东西是必须写的，比如 #！/bin/bash …………. 符号#！是用来告诉系统它后面的参数是用来执行该文件的程序。此处我们是使用的bash来执行程序的。 2、注释 “#“ 此为Shell的注释符号，以 # 开头的句子表示注释，直到这一行结束。 就像C/C++中的 ”// “一样，起到注释程序的作用，本人觉得注释在一个程序和Shll中是必不可少的，这样有利于日后的阅读。 3、变量 在Shell编程中，所有的变量都是由字符串组成，并且不需要对变量进行声明。 在Bourne Shell有如下4钟变量： （1）用户自定义变量； （2）位置参数即命令行参数； （3）预定义变量 （4）环境变量 3.1 用户自定义变量 3.1.1 Shell变量名的命名规则同C++的命名规则；（由数字、下划线和字母组成，首字母不能为数字，同时中间不能有空格，也不能有其他的标点符号） 3.1.2 变量的赋值 变量名=值 #变量赋值时，符号两边都没有空格 COUNT=1 #变量名通常使用大写，方便识别 3.1.3 变量的调用 要取用一个变量的值，只需在变量名的前面加一个$，比如： #!/bin/bash #对变量赋值： VAR=“hello world” #等号的两边俊没有空格存在 #打印变量VAR的值： echo ”VAR is ： “$VAR Linux Shell/bash 从右向左赋值，比如： #!/bin/bash Y=y X=$y echo X=$x 对于变量名与其他文字混淆的情况，使用花括号来告诉我们要打印的是某个变量，比如： num=2 echo “This is the ${num}nd” #${num}nd 3.2 预定义变量 $# 的含义是包含命令行参数的个数，命令参数不计，即只从$1开始计数。 4、Shell程序和语句 4.1 算术运算命令……
