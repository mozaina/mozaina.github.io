解决shell脚本报错bashxxxbinbashMbadinterpreterNosuchfileordirectory

2020年9月2日
问题 在执行shell脚本时出现“-bash: xxx: /bin/bash^M: bad interpreter: No such file or directory”的错误 原因分析 主要原因是test.sh是我在windows下编辑然后上传到linux系统里执行的。.sh文件的格式为dos格式。而linux只能执行格式为unix格式的脚本。 解决方法 1.首先用vi或者vim命令打开shell文件 2.在命令模式中输入:set ff，回车后可以看到文件的格式为dos ~ :set ff ~ fileformat=dos 3.修改文件格式为unix 此处有三种方法修改 方法一：打开文件使用vi或vim命令修改 命令如下： :set ff=unix 执行完后再通过“:set ff”命令查看文件格式，结果如下： fileformat=unix 方法二：直接使用dos2unix命令修改 命令如下： dos2unix xxx.sh 其中xxx为shell文件名 修改完再次执行shell文件就能正确运行了。 方法三： 直接执行sed -i “s/\r//” a.sh来转化， 然后就可以执行./a.sh运行脚本了。……
