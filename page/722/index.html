2020年9月2日
参考地址：https://ruby-china.org/topics/33843 （1）> gem sources –add http://gems.ruby-china.org 遇到问题： Error fetching https://gems.ruby-china.org/: certificate verify failed (https://gems.ruby-china.org/specs.4.8.gz) （2）定位ruby gem 证书存放的位置： gem which rubygems C:\jruby-9.1.2.0\lib\ruby\stdlib\rubygems.rb 然后就知道要去C:\jruby-9.1.2.0\lib\ruby\stdlib\rubygems\ssl_certs （3）下载新证书： http://curl.haxx.se/ca/cacert.pem 放到C:\jruby-9.1.2.0\lib\ruby\stdlib\rubygems\ssl_certs目录下 配置环境变量： SSL_CERT_FILE 值为刚下载证书的存放位置：C:\jruby-9.1.2.0\lib\ruby\stdlib\rubygems\ssl_certs\cacert.pem （4）打开新的cmd窗口 gem sources –add http://gems.ruby-china.org 成功！ 【后续】 20180830发现网站修改域名，将org改成com，所以cmd执行下面操作 C:\Users\wangxue>gem sources *** CURRENT SOURCES *** http://gems.ruby-china.org https://gems.ruby-china.org C:\Users\wangxue>gem sources http://gems.ruby-china.org *** CURRENT SOURCES *** http://gems.ruby-china.org https://gems.ruby-china.org C:\Users\wangxue>gem sources -r http://gems.ruby-china.org http://gems.ruby-china.org removed from sources C:\Users\wangxue>gem sources -r https://gems.ruby-china.org……
