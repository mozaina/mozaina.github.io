ruby连接mysql问题处理办法汇总

2020年9月2日
1 安装mysql,这里有个版本的问题，其实在xp下面，使用2.8.1 x86-mingw32（mysql-2.8.1-x86-mswin32.gem）是没有问题的，如果一定要安装 x86-mswin32平台的，那就在这样安装gem install mysql –platform x86-mswin32.同样，安装其它模块，只要加上 —platform参数，就会安装你想要的版本。 2.安装成功以后，用require ‘mysql‘ 时提示“in `require’: no such file to load – mysql (LoadError)” 这个问题就是lib\ruby\site_ruby\1.8\i386-msvcrt没有mysql.so文件，不知道为什么我安装mysql时没有提示选择安装平台，也没有在相关文件下发现mysql.so文件，那个mysql_api.sq根本不起作用。 这个文件已经有人编译好了，可以在 http://www.blogjava.net/cap/archive/2006/01/20/28731.html找到（没能打开连接）。 这个问题还有一个解决方法： 在require’mysql’这前先require’rubygems’; 3. ruby-mysql乱码问题 我的数据库是utf8编码的，dos下中文显示出来全是乱码，想想php在处理这类问题的办法，不是什么将数据库改编码，而是在sql前面里加一条“SET NAMES GBK;”，这个办法在ruby里也实用。 也许在你那时要用“SET NAMES UTF8;”。……
