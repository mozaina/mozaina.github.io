2020年9月2日
首先，2012年了，已经有了Ruby 1.9.3，为什么我要安装1.9.2？ 答案：因为我在学习Ruby on Rails，参考书Ruby on Raisl 3 Tutorial是根据1.9.2写的；而且heroku服务器看样子目前只支持Ruby 1.9.2 启动Ubuntu，通过以下命令安装一些相关的库 $ sudo apt-get install gcc g++ build-essential libssl-dev libreadline- gplv2-dev zlib1g-dev linux-headers-generic 下载最新的（或者合适的）Ruby 1.9.2源码 查看：ftp://ftp.ruby-lang.org/pub/ruby/1.9/ 或者查看：http://ftp.ruby-lang.org/pub/ruby/1.9/ $ wget ftp://ftp.ruby-lang.org/pub/ruby/1.9/ruby-1.9.2-p290.tar.gz 编译安装Ruby $ tar -zxf ruby-1.9.2-p290.tar.gz $ cd ruby-1.9.2-p290 $ ./configure –prefix=/usr/local/ruby $ make && sudo make install 环境设置 $ sudo vi /etc/environment 将Ruby的路径加入环境变量 中并保存/etc/environment，如下面内容： PATH=“/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/ruby/bin” 修改了环境变量文件后，需要通过source命令让修改马上生效，命令如下： $ source /etc/environment 为了检查安装是否成功，可以输入下面的命令进行测试 ： $ruby –v 确认安装成功后通过一下命令添加命令链接，目前我也不清楚创建这些链接的目的是什么，按照Ruby“约定大于配置”的原则，应该是一种约定。(keyboardota) $ sudo ln -s /usr/local/ruby/bin/ruby /usr/local/bin/ruby……
