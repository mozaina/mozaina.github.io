Shell的编程过程 Shell独立与内核，它是连接内核和应用程序的桥梁，并由输入设备读取命令，再将其转为计算机可以理解的机械码，Linux内核才能执行该命令。 Shell是一种应用程序，当用户登陆Linux系统时，Shell就会被调入内存去执行。 Shell提供了用户与内核进行交互操作的一种接口，接收用户输入的命令，并把他送入内核去执行。 在Linux中，Shell不但能与用户进行逐个命令交互执行，也可以通过脚本语言进行编程。 通过对Shell进行编程，可使大量任务自动化，对于系统管理、系统维护方面特别有用。 Shell脚本的基本结构 Shell脚本在不同的Linux系统都能运行 可使用多种编辑器，如vim等 Shell脚本通常是以 .sh 为后缀 第一行要以 #! /bin/bash 开头 ，代表该文件使用的是bash语法 # 表示注释，后面跟一些该脚本相关的的注释内容以及作者、创建日期、版本等。 演示操作： 使用vim编辑器 新建文件first.sh vim first.sh 按 i 进入编辑模式 第一行是要求要有的，作用：定义执行该shell的程序在哪，不加就系统默认： #/bin/bash 第二行是注释： #this is a shell data命令显示时间，echo命令显示字符串： date echo "hello,word!" 按Esc键切换到命令模式，输入 :wq 保存退出 操作演示：让Shell脚本运行 给Shell文件赋予可执行的权限： chmod a+x first.sh 检查权限： ll first.sh 显示first.sh已有可执行权限(x) 下面执行： 方法一： ./first.sh Shell脚本正常运行 方法二：通过Shell命令来执行Shell文件 sh first.sh……
