CCC与Shell的如何互传信息

2020年9月2日
一、引子 在很多时候，我们期望C程序与Shell能够互传信息。比如：我们需要用Shell来控制C程序的执行，并且希望C程序能从Shell中获取相关信息；同样，我们也需要从C程序中返回一些信息给Shell，以便Shell根据返回信息执行下一步操作。 二、Shell传递信息给C程序 熟悉Linux下C编程的人都知道，C语言中”main”函数有与Shell命令的接口： int main(int argc, char **argv) { // body of main } 其中，”argc”(argument counts)表示传入参数的个数，”argv”(argumen values)表示传入参数的值。 来看下面一段代码： #include int main(int argc, char **argv) { printf(" argc = %d \n", argc); printf(" argv0 = %s \n", argv[0]); printf(" argv1 = %s \n", argv[1]); printf(" argv2 = %s \n", argv[2]); return 0; } 将其编译链接为可执行文件”./exchange_msg_shell”，并且在shell命令行执行： ./exchange_msg_shell hello world 其输出如下： argc = 3 argv0 = ./exchange_msg_shell argv1 = hello argv2 = world 参数”argc”存储了传入参数的个数，参数”argv”以数组的形式存储了传入参数的值，并且第一个值”argv[0]“表示该可执行文件的文件名，”argv[1], argv[2]“依次表示后面紧跟的参数值。……

CDH中hue执行shell脚本调用sqoop脚本报错 1、Could not load db driver class: com.oracle.jdbc.Driver 解决：oracle驱动不存在。你可以把驱动包拷贝到/opt/cloudera/parcels/CDH/lib/sqoop/lib 。看到这个错误的时候，需要提示的一点，服务器上执行sqoop脚本。不会报错，但是在CDH中就会报错。因为你需要在CDH的lib包中添加驱动。 2、Import failed: org.apache.hadoop.security.AccessControlException: Permission denied: user=yarn, access=WRITE, inode="/user/yarn":hdfs:supergroup:drwxr-xr-x 出现这个错误我也查询了很多。修改目录权限，（其实这是最不好的）。还有一种要你关闭目录检查（CDH启动会直接报错）。 解决： 在HDFS的配置，找到hdfs-site.xml安全阀。添加dfs.permissions 为false (切记要到安全阀这里设置) 虽然这也是关闭hdfs检查，但是不会配置重启报错。……
