couldnotopeninputfilelinux编写shell遇到了小问题

2020年9月2日
对于linux新人来说，这个问题是天大的。但最后还是被我的耐心给解决掉了。 是这样的 今天学习秒杀知识点，其中需要用到定进任务，所以我就启动了一年前装好的虚拟机。对于linux，本人还是有点基础知的。 于是信心满满的把在windows平台上写好的PHP文件上传到linux。跑了起来，结果呢，老是出现 could not open input file 。于是乎按在windows平台上的思维，肯是文件路径出错，然后就满世界的去查脚本。但一无所获。 于是把这个错误放到百度一查，终于有点收获。是什么文件系统格式，于是就给处理订单的文件 set ff=unix 下，再跑。还是一样问题。那问题来了，是不是脚 本也要呢? 后来索性再给脚 本 set ff下，结果还是不行。 于是单独用 下，发现有个错误，就是一个字段少了个字母，赶紧用vi命令改好，再PHP 跑，还是不行。于是跑去看脚本，发现没有用cd 定位到脚本所有的文件（这里有思维错误了，以为脚本与goods.php文件同在一目录下，就不用cd 定位）。后来加上cd 到脚本文件。一跑，跑起来了。 总结：1、文件格式错误。 2、脚本的位置要绝对指明，因为crontap 是在bash中运行。 结尾 ，交了很多智商税。。。。……
